APIScout 使用指南

从安装到出文档，5 分钟搞定

一、安装

方式 A：你的电脑有 Python（Mac/Linux/Windows）

$ pip install playwright genson pyyaml jinja2 click fastapi uvicorn
$ playwright install chromium
$ cd apiscout && pip install -e .
# 完成！

方式 B：同事的 Windows 电脑没有 Python（在线安装）

电脑能联网的情况下，右键用 PowerShell 运行：

1
右键 pack/setup_portable.ps1 → 用 PowerShell 运行

或在 cmd 中：powershell -ExecutionPolicy Bypass -File pack\setup_portable.ps1

自动下载 Python 3.11 + pip + 所有依赖 + Chromium 浏览器引擎，约 5 分钟

2
完成后双击生成的「启动APIScout.bat」

Web 面板自动启动，浏览器自动打开

方式 C：同事的 Windows 电脑没有 Python（离线安装，无需联网）

1
U 盘拷贝 apiscout 项目文件夹

约 230MB，包含 Python、Chromium、所有依赖

2
双击 pack/portable/install_offline.bat

自动解压 Python + 安装依赖 + 解压浏览器引擎，约 1 分钟

3
双击生成的「启动APIScout.bat」

Web 面板自动启动，浏览器自动打开

二、使用方式（推荐：Web 面板）

1
启动 Web 面板
$ apiscout web
APIScout Web 面板: http://localhost:9527

浏览器自动打开控制页面

2
输入目标系统 URL，点击「开始扫描」

输入客户系统的地址，例如 https://eam.customer.com

Playwright 浏览器窗口弹出

3
在弹出的浏览器中登录系统

输入用户名密码登录。APIScout 不存储密码，只借用登录后的 session。

安全提示：APIScout 只录制网络请求，不修改任何数据，不点击任何危险按钮。
4
正常操作系统

像平时使用系统一样操作：点菜单、看列表、查详情、搜索。每个操作触发的 API 都会被自动录制。

Web 面板实时显示已捕获的请求数量。

技巧：尽量多点不同的功能模块。每个菜单点一遍，覆盖率最高。
5
点击「停止录制 → 生成文档」

APIScout 自动分析录制的数据：

  • 识别所有 API 端点
  • 推断请求/响应 schema
  • 检测认证方式
  • 生成 OpenAPI spec + Swagger UI 交互文档
6
查看结果

点击页面上的链接查看生成的文档。最重要的是 Swagger UI 文档 — 交互式 API 文档，可以看到所有端点、参数、响应格式。

六、常见问题

Q: 抓不到 API 请求？

可能原因：前端和后端在不同端口/域名。APIScout 已改为黑名单模式，会录制所有 XHR/fetch 请求。如果仍抓不到，用 --debug 参数查看详细日志。

Q: 系统用了 Vaadin/JSF 等服务端渲染框架？

浏览器不发 REST API 请求，抓包抓不到。APIScout 会自动检测框架并尝试使用适配器（如 Jmix metadata）。如果没有适配器，可以走数据库 schema 路径（V1.1）。

Q: 会不会修改客户数据？

不会。APIScout 只录制网络请求（被动监听），不发送任何请求、不修改任何数据。自动探索模式有安全护栏，不点击"删除/提交/保存"等按钮。手动模式下完全由你控制操作。

Q: 支持哪些技术栈？

前后端分离（Vue/React + REST API）完美支持（抓包主路径）
若依 / JeecgBoot完美支持（适配器 + 抓包）
Spring Boot完美支持（下载 Swagger + 抓包）
Jmix完美支持（metadata 适配器）
Vaadin / JSF / Wicket适配器模式（浏览器抓包无效）
传统 MVC（jQuery/JSP/PHP）支持（如果有 XHR 调用）
最佳实践：不管什么系统，先用 apiscout scan URL --manual 试一遍。登录后点几个菜单，看能抓到多少 API。抓到就行，抓不到再换路径。
