1这工具帮你做什么

一句话：不管客户系统是什么技术栈、有没有 API 文档，APIScout 都能帮你快速产出一份接入方案。客户给什么，你用对应的玩法：

客户系统的情况你怎么做产出
有 REST API，但没文档打开浏览器，手动点一遍要接入的功能（录制）OpenAPI 规格书 + Swagger 文档
有 Swagger / OpenAPI 文件或地址直接导入统一格式的文档
是 Jmix / Spring Boot / 若依框架自动识别框架并生成完整 CRUD 接口规格
能拿到数据库只读账号扫描数据库结构表结构 + 枚举 + 关系报告
核心理念 手动录制 + 人工审核是主线，自动探索只是辅助。别指望全自动出完美结果——你点过的功能，它才抓得到。

2安装（三种场景，挑一个）

场景 A：你的电脑已经装了 Python 最常用

要求 Python 3.11 或更高。在项目根目录（含 pyproject.toml 那一层）打开终端/命令行：

# 1. 建议先建独立环境（可选，但推荐）
$ python -m venv .venv
$ source .venv/bin/activate # Windows 用：.venv\Scripts\activate

# 2. 安装本工具（依赖会自动装齐，含 Web 面板）
$ pip install -e .

# 3. 下载浏览器内核（必须，约 150MB，只需一次）
$ playwright install chromium
# 完成！
要扫数据库再加装驱动 按目标库类型选装：
pip install -e ".[postgresql]" ／ ".[mysql]" ／ ".[oracle]" ／ ".[mssql]" ／ ".[db-all]"（全装）
要用 AI 增强：pip install -e ".[ai]"

场景 B：客户/你的 Windows 电脑没装 Python（在线安装）

1
把整个 apiscout 项目文件夹拷到目标电脑（U 盘即可）
2
右键 pack/setup_portable.ps1 → 用 PowerShell 运行

或在 cmd 里执行：powershell -ExecutionPolicy Bypass -File pack\setup_portable.ps1

会自动下载 Python + 所有依赖 + Chromium，约 5 分钟（需联网）。

3
完成后双击生成的「启动APIScout.bat」即可

场景 C：Windows 电脑没网、没 Python（离线包）

用 pack/portable/ 离线便携包：拷到目标电脑后双击 pack/portable/install_offline.bat，全程不需要联网。适合客户内网隔离环境。

提示离线包较大（约 229MB），需提前在有网的电脑上准备好。详见 pack/README_部署.md。

3启动系统

装好之后，推荐用 Web 面板（最直观，适合不熟命令行的同事）：

$ apiscout web
APIScout Web 面板: http://localhost:9527

然后浏览器打开 http://localhost:9527。换端口用 apiscout web -p 8000。

用便携/离线包安装的，直接双击 pack/run.bat 或「启动APIScout.bat」，会自动起服务并打开浏览器。

4核心流程：录制式抓 API

这是最常用、最稳的玩法——你在浏览器里操作客户系统，APIScout 在背后默默记录所有接口请求，操作完一键生成文档。

1
在 Web 面板输入客户系统的网址，点「开始扫描」

会自动弹出一个浏览器窗口。

2
在弹出的浏览器里，正常登录客户系统

账号密码照常输，登录态会被一起记录。

3
把你要接入的功能，挨个手动点一遍

查列表、看详情、翻分页、做筛选……点得越全，抓到的接口越全。你没点到的功能，是抓不到的。

4
操作完，回到 Web 面板点「停止录制 → 生成文档」
5
查看产物

在 output/<客户域名>/ 目录下，打开 api_docs.html 就是一份可交互的 Swagger 文档。

安全护栏 自动探索模式不会去点「删除 / 提交 / 保存」等危险按钮，不会修改客户数据。但稳妥起见，第一次跑建议用纯手动模式（见下方命令行 --manual），完全由你控制点哪里。

5命令行速查

不想用 Web 面板、或要批量处理时，用命令行更快。全部命令：

命令作用常用例子
web启动 Web 面板apiscout web -p 9527
scan打开浏览器扫描一个网站apiscout scan https://eam.客户.com --manual
import导入已有 Swagger/OpenAPIapiscout import https://客户.com/v3/api-docs
analyze重新分析已抓的数据apiscout analyze output/客户/capture.jsonl
db扫描数据库结构apiscout db "postgresql://..."
enrich用 AI 补全字段语义apiscout enrich output/客户 --api-key sk-xxx

scan 常用参数

  • --manual — 纯手动模式，不自动探索，只录你点的操作（最稳，推荐首选）
  • -o 目录 — 指定输出目录（默认用域名命名）
  • --append — 追加模式，多次扫描累积到同一份数据
  • --title "xxx" — 自定义 API 文档标题
多次扫描累积 一次没点全没关系，加 --append 再扫一遍，新抓的接口会合并进同一份文档。

6数据库扫描

客户能给数据库只读账号时，可以直接扫库结构——拿到所有表、字段、枚举值和表间关系。

# 基本用法
$ apiscout db "postgresql://readonly:pass@192.168.1.100:5432/eam"

# 排除框架自带的无关表（工作流、定时任务等），结果更干净
$ apiscout db "postgresql://..." -x "act_*" -x "qrtz_*" -x "platform_*"

# 用数据库结构反过来增强已抓的 OpenAPI 文档（更精确的字段类型/枚举）
$ apiscout db "postgresql://..." --enrich output/客户/openapi.yaml
参数说明
-x, --exclude排除表名，支持通配符（如 act_*），可重复多次
--enrich 路径用库结构交叉增强已有的 OpenAPI 文档
--dialect数据库类型：postgresql / mysql / oracle / mssql（一般能自动识别）
--host / --port / -u / -p / -d不想拼连接串时，分开传主机/端口/用户/密码/库名

支持 PostgreSQL、MySQL、Oracle、SQL Server。产物是一份 HTML 结构报告，可直接发客户审核确认。

务必用只读账号扫库只读 information_schema，但请始终向客户索要只读权限账号，避免任何误操作风险。

7看懂输出产物

每次扫描的结果都在 output/<客户域名>/ 下：

文件是什么给谁看
api_docs.html可交互的 API 文档（Swagger UI）直接发客户确认
draft_spec.yamlOpenAPI 3.1 规格书给 insight68 接入用
auth_profile.yaml认证方式档案（登录怎么做的）实施时配认证用
report.html覆盖率报告（抓到多少接口）自检 / 给客户看进度
capture.jsonl原始捕获数据含客户业务数据，内部留存，勿外传

8实施最佳实践

  • 手动优先：第一次接触客户系统，用 --manual 或 Web 面板自己点，最稳、最可控。
  • 点得全才抓得全：把所有要接入的业务功能都走一遍——列表、详情、新增、编辑、查询、分页。
  • 结果一定要人工审核：自动推断出的字段类型、枚举、命名，发客户前自己先过一遍。纯自动 = demo 质量。
  • 看覆盖率报告：report.html 告诉你抓到了多少，漏的功能补扫一遍（--append）。
  • 有库就扫库：能拿到数据库只读账号时，db --enrich 能让接口文档的字段精度上一个台阶。

9常见问题排错

Q：跑 apiscout 提示「命令不存在」？

说明没装好或没在对的环境里。确认在项目根目录执行过 pip install -e .；如果建了虚拟环境，每次开终端要先 activate。

Q：启动浏览器报错 / 提示找不到 Chromium？

少跑了一步：playwright install chromium（下载浏览器内核，需联网）。

Q：抓不到 API 请求，文档是空的？

几种可能：① 你登录后没真正去点功能（只抓你点过的）；② 系统是 Vaadin/JSF 等服务端渲染框架，前端不走 REST 接口——这种没法靠录制抓，得走数据库扫描或框架适配器；③ 接口在别的子域名，多点几个页面让它跳转过去。

Q：扫数据库报「缺少驱动」？

按库类型补装：pip install -e ".[postgresql]"（或 mysql/oracle/mssql）。

Q：会不会改坏客户的数据？

不会。录制只是旁观记录；自动探索有护栏，不点删除/提交/保存类按钮；数据库扫描只读结构。但请坚持用只读数据库账号，并优先用手动模式。

Q：跑测试确认环境没问题？

pip install -e ".[dev]" 后执行 pytest tests/ -q，应看到 197 passed（纯本地 mock，不连任何真实系统）。

10安全红线（必读）

客户数据不外传 capture.jsonl、output/、test_output/ 里都是客户真实业务数据。内部留存即可，不要提交到代码仓库、不要发到群里、不要传给无关方。这些目录已默认在版本控制里忽略。
  • 不要在代码或文档里硬编码客户密码、密钥；AI 的 API Key 通过参数或环境变量传，用完不要写进文件。
  • 数据库一律用只读账号。
  • 给客户的产物只发 api_docs.html / report.html 这类整理过的文档，原始捕获数据不外发。
